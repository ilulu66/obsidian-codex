这份时间线按用户反馈顺序整理，方便写项目复盘时追溯产品演进。

## 1. 初始需求：在 Obsidian 里调用 Codex

用户原话：

> 我现在在用 obsidian+claudian，但是没有 codex 的插件，在 obsidian 就不方便调用你，你能做一个插件吗，参考 claudian 的做法

执行：

- 查看本地 Claudian 插件结构。
- 识别 Claudian 的核心模式：Obsidian 面板、命令入口、vault 工作目录。
- 创建 Codex 插件 MVP。

产出：

- `.obsidian/plugins/codex/manifest.json`
- `.obsidian/plugins/codex/main.js`
- `.obsidian/plugins/codex/styles.css`

## 12. 更好的运行状态、恢复和图片

用户原话：

> 更好的运行状态，比如显示：正在读取上下文、正在调用 Codex、已运行多少秒、可停止、失败原因。这个状态我希望是能看到的。现在这个插件我发现没办法增加窗口，好像被重命名挡住窗口。另外就是好像没有在桌面端看到这个对话，我有办法恢复吗？如果我关掉了窗口。另外就是输入框现在不支持富文本，没办法插入图片。

执行：

- 增加可见状态面板：阶段、耗时、停止按钮、失败原因。
- 顶部增加 `+`，支持继续新增窗口，最多 20 个。
- 窗口标签栏改为横向滚动，避免长名字挡住按钮。
- 增加恢复命令：`Codex: Restore Codex panel / 恢复 Codex 面板`。
- 支持图片按钮、粘贴图片、拖拽图片到输入框，并通过 Codex CLI 的 `-i` 参数传给 Codex。
- 在 README 里说明：Obsidian 面板会本地保存记录，但这些对话不一定出现在 Codex 桌面端侧栏。

发布：

- v0.4.0
